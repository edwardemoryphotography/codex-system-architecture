# Skill: project bridge (Notion)

When a repo lives only in **GitHub Repo Audit Tracker** (Command Center) but has **no Projects row**, run this once.

## Steps

1. Find audit row (search repo name).
2. Create **Projects** row: `Name`, `Status` = In progress, link doctrine pages in body.
3. Create **Tasks** from `Action Required` bullets; set `Projects ` relation to new row URL.
4. Replace audit page body: link to Projects row + repo + next ship.
5. Set audit `Status` → In Progress.
6. Add **notion-wiki skills index** link on LLM Wiki hub if missing.
7. Update **Corpus map** active projects with Projects row URL.
8. Append **Wiki Log**.

## When not needed

Project already in Projects DB with Tasks linked — use `project-context.md` only.
